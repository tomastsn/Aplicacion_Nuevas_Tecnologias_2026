## 1. Amazon SageMaker
### Problema que resuelve con ML
Es una plataforma integral que permite a empresas y desarrolladores construir, entrenar e implementar modelos de Machine Learning rápidamente. Resuelve la complejidad de gestionar la infraestructura subyacente, permitiendo automatizar el ciclo de vida completo del ML (desde la preparación de datos hasta el despliegue).

* **Tipo de ML:** **Supervisado** (principalmente).
* **¿Por qué este tipo?:** Aunque es una plataforma que soporta varios tipos, su núcleo operativo se basa en algoritmos que aprenden a partir de datos etiquetados para realizar tareas de clasificación y regresión con alta precisión.
* **Datos necesarios:** * Datos estructurados y etiquetados (ej. historial de ventas).
    * Logs de rendimiento y métricas en tiempo real.
    * Conjuntos de datos de entrenamiento, validación y prueba.

## 2. Netflix (Motor de Recomendaciones)
### Problema que resuelve con ML
Resuelve el problema de la "parálisis por elección". El sistema predice qué contenido (películas/series) es más relevante para cada usuario, personalizando la interfaz y aumentando significativamente la retención de clientes.

* **Tipo de ML:** **Supervisado** (con técnicas de filtrado colaborativo).
* **¿Por qué este tipo?:** El algoritmo utiliza el feedback histórico del usuario (lo que vio, lo que calificó con "pulgar arriba") como etiquetas para predecir la probabilidad de que le guste un contenido nuevo.
* **Datos necesarios:** * Historial de visualización y tiempo de permanencia en un título.
    * Valoraciones explícitas (likes) e implícitas.
    * Metadatos del contenido (género, actores, año de estreno).

## 3. Uber (Predicción de Precios y ETA)
### Problema que resuelve con ML
Optimiza el emparejamiento entre conductores y pasajeros. Utiliza ML para calcular el tiempo estimado de llegada (ETA) y el sistema de "precios dinámicos" que se ajusta según la oferta y la demanda en tiempo real.

* **Tipo de ML:** **Supervisado**.
* **¿Por qué este tipo?:** Utiliza modelos de regresión entrenados con millones de viajes anteriores donde ya se conoce el tiempo real que tardó el trayecto y el precio final, permitiendo predecir estos valores para viajes futuros.
* **Datos necesarios:** * Historial de coordenadas GPS de viajes previos.
    * Datos de tráfico y clima en tiempo real.
    * Densidad de usuarios activos y conductores disponibles por zona.

## 4. Spotify (Descubrimiento Semanal / Daily Mix)
### Problema que resuelve con ML
Analiza los hábitos de escucha para curar listas de reproducción personalizadas. Esto ayuda a los usuarios a descubrir nuevos artistas sin tener que buscarlos manualmente, manteniendo el compromiso con la plataforma.

* **Tipo de ML:** **Híbrido (Supervisado y No Supervisado)**.
* **¿Por qué este tipo?:** Usa **No Supervisado** (Clustering) para agrupar a usuarios con gustos similares y **Supervisado** para clasificar canciones según sus atributos técnicos y predecir si encajan en el perfil del usuario.
* **Datos necesarios:** * Historial de reproducción (canciones escuchadas, saltadas o guardadas).
    * Análisis de audio (tempo, tono, "bailabilidad").
    * Comportamiento de otros usuarios con perfiles similares.

## 5. Mercado Libre (Búsqueda y Recomendaciones)
> **Nota:** Producto de origen Latinoamericano (Argentina).

### Problema que resuelve con ML
Optimiza la relevancia de los resultados de búsqueda y personaliza las recomendaciones de compra. También utiliza ML para la detección de fraude en Mercado Pago y para la clasificación automática de productos dentro de las miles de categorías del sitio.

* **Tipo de ML:** **Supervisado y No Supervisado**.
* **¿Por qué este tipo?:** Usa modelos **supervisados** para clasificar productos y predecir la intención de compra, y modelos **no supervisados** para segmentar usuarios según su comportamiento de navegación sin etiquetas previas.
* **Datos necesarios:** * Historial de clics y términos de búsqueda.
    * Atributos de los productos (título, descripción, precio).
    * Datos transaccionales y de comportamiento financiero para prevención de fraude.
